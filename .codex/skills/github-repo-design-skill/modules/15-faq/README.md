# 常见问题

## Q: README 应该多长？

**A:** 取决于项目复杂度。小型工具库保持简洁（一屏），大型框架可以更详细。关键是让用户快速找到需要的信息。

## Q: 应该用什么许可证？

**A:**
- 希望最大程度开放：MIT 或 Apache 2.0
- 希望衍生作品也开源：GPL-3.0
- 企业项目：Apache 2.0（有专利保护）

## Q: 如何提高仓库可见度？

**A:**
1. 添加相关 Topics
2. 写好 README 和描述
3. 保持活跃更新
4. 参与社区讨论
5. 在社交媒体分享

## Q: Issue 模板用 YAML 还是 Markdown？

**A:** 推荐 YAML 格式（.yml），支持表单验证和下拉选择，用户体验更好。

## Q: 什么时候需要 CONTRIBUTING.md？

**A:** 当你希望接受外部贡献时。即使是小项目，清晰的贡献指南也能帮助潜在贡献者。

## Q: 如何处理敏感信息？

**A:**
1. 使用 `.gitignore` 排除敏感文件
2. 使用环境变量存储密钥
3. 启用 GitHub Secret Scanning
4. 使用 `.env.example` 提供模板

## Q: 分支保护规则会影响效率吗？

**A:** 短期可能增加一些步骤，但长期能：
- 防止意外推送到主分支
- 确保代码质量
- 保持提交历史清晰

## Q: 需要 Wiki 还是 docs 目录？

**A:**
- **Wiki**: 适合社区驱动的文档，任何人可编辑
- **docs 目录**: 适合版本控制的文档，与代码同步

## Q: 如何选择 CI/CD 工具？

**A:**
- **GitHub Actions**: 与 GitHub 深度集成，推荐首选
- **CircleCI**: 复杂工作流，并行执行
- **Travis CI**: 开源项目免费
- **GitLab CI**: GitLab 用户首选

## Q: Monorepo 还是 Multi-repo？

**A:**
- **Monorepo**: 代码共享多、团队协作紧密
- **Multi-repo**: 项目独立、团队分散、权限隔离需求
