# 安全策略

## 支持范围

当前仓库主要用于课程项目开发与展示。原则上，仍在维护中的最新版本会继续接收安全修复。

## 漏洞报告方式

如果你发现了潜在安全问题，请不要直接在公开 Issue 中披露完整漏洞细节。

建议处理方式：

1. 先整理复现步骤、影响范围和风险等级。
2. 通过团队内部私有沟通渠道联系项目维护者。
3. 在修复完成并确认风险可控后，再决定是否公开记录。

## 报告内容建议

- 受影响模块或接口
- 复现步骤
- 影响范围
- 可能的修复方向

## 常见敏感信息

以下内容不应提交到仓库：

- 数据库真实连接串
- 第三方服务 API Key
- 生产环境账号或密码
- 含个人隐私的测试数据

当前仓库已通过根目录 `.gitignore` 忽略本地 `.env`、虚拟环境和常见构建产物，但提交前仍建议手动复查。
